## 1. Documentation → science consistency issues found

### Issue 1 (MINOR): `multicase.pt` dataset version label in `project_status.md`

- **File:** `docs/project_status.md`, line 31.
- **Claim:** `Tier 3 PINN multicase (v2, 8-channel) | outputs/tier3_pinn_multicase.*`
- **Actual:** `outputs/tier3_pinn_multicase_log.md` shows the checkpoint was trained on the **v1** dataset (3 cases: A, B, C; 36 total samples; 24 train + 12 val). The "(v2, 8-channel)" label could be read as "trained on the v2 dataset" (5 cases, 60 samples), which is incorrect.
- **Impact:** LOW. The `multicase.pt` checkpoint is a milestone artifact from the v1 round. All generalization claims are based on the rotation studies (`rotation_holdout{A..E}` trained on v2, `rotation_v3_*` on v3, `rotation_v4_*` on v4), which correctly identify their dataset versions. The multicase checkpoint's log is internally honest ("With only two training cases…"). No scientific claim is overstated.
- **Suggested correction:** Change "(v2, 8-channel)" to "(v1 dataset, 8-channel architecture)" or "initial multicase milestone (v1 dataset, 8-channel)" to avoid ambiguity.

### Issue 2 (COSMETIC): `generate_figures.py` only covers fig01–fig07

- **File:** `scripts/generate_figures.py` header (line 14).
- **Claim:** *"Regenerate every figure referenced by the technical report and the slides."*
- **Actual:** fig08–fig15 are generated by four other scripts (`analyze_rotation_and_variance.py`, `analyze_prf_and_seed_variance_v2.py`, `validation_prep.py`, `compare_picmcc.py`, `make_pressure_axis_figure.py`).
- **Impact:** NONE for correctness — each figure has a verified generator. But a reader running only `generate_figures.py` will get fig01–07 and not fig08–15.
- **Suggested correction:** Add a comment to `generate_figures.py` noting that fig08–15 are produced by the analysis/validation scripts listed in `docs/reproducibility.md`, or add a wrapper script that calls all generators in sequence.

### Issue 3 (INFO): `outputs/validation_prep_report.md` is shorter than expected

- **File:** `outputs/validation_prep_report.md` (50 lines).
- **Context:** The same file in `SF6_tier3_repo 5` was 86 lines. The difference is not content loss — repo 8's version reflects the newer `scripts/validation_prep.py` that outputs a more concise "honest scope" format, with the exhaustive mapping table now living in `docs/project_status.md` instead.
- **Impact:** NONE. No information was lost; it was restructured.

### No other documentation-science mismatches found.

All other reference-to-artifact links verified:
- `project_status.md` headline numbers (ΔTe ≈ 0.23, hold-out B 1.114 ± 0.008, hold-out C 0.589 ± 0.058, seed noise 10–20%) traceable to their cited JSON/MD sources.
- `README.md` figure references (fig01–fig15) match existing PNGs.
- `future_work_completed.md` references to v2/v3/v4 datasets, rotation tables, and PRF tables all point to existing files.
- `report/technical_report.tex` and `slides/tier3_presentation.tex` collectively contain 129 and 25 references to the key artifacts.
